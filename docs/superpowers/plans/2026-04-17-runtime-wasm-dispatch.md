# Runtime WASM Dispatch — Plan 4B.0

> **Prerequisite for Plan 4B (designer refactor).** Adds actual wasmtime Component dispatch to `greentic-ext-runtime` so designer can call `runtime.invoke_tool(ext_id, "validate_card", args)` end-to-end.

**Goal:** `ExtensionRuntime` can load a WASM Component, instantiate it with a `Linker` that fulfills the host imports (logging / i18n / secrets / broker / http), and call exported WIT functions from Rust code. Validated end-to-end by calling the `greentic.adaptive-cards` extension's `list_tools` + `invoke_tool` from an integration test.

**Tech:** wasmtime 43 Component Model + wit-bindgen host-side bindings (`bindgen!` macro) + tokio async runtime.

**Tasks:** 5 total, one per phase.

---

Note: the real extension world (e.g. AC's `design-extension`) imports host + exports base + design sub-WITs. Our runtime side imports the design WIT to CALL its exports, and provides host traits to fulfill its imports.

Actually simplify: generate bindings against a **synthetic runtime-side world** that imports what extensions export + exports what extensions import. We'll define that inline:

Create `wit/runtime-side.wit`:

```wit
package greentic:runtime-side@0.1.0;

world runtime-side {
  // We (runtime) import what extensions EXPORT:
  import greentic:extension-base/manifest@0.1.0;

  import greentic:extension-design/validation@0.1.0;
  import greentic:extension-design/prompting@0.1.0;
  import greentic:extension-design/knowledge@0.1.0;

  // We (runtime) export what extensions IMPORT (host traits):
  export greentic:extension-host/logging@0.1.0;
  export greentic:extension-host/i18n@0.1.0;
  export greentic:extension-host/secrets@0.1.0;
  export greentic:extension-host/broker@0.1.0;
```

## Task 2 — Host state + Linker setup

**Files:**

- [ ] **Step 1: Write `HostState` — fulfills all 5 host import interfaces**

```rust
use crate::host_bindings::greentic::extension_host;

pub struct HostState {
    pub extension_id: String,
    pub permissions: greentic_ext_contract::describe::Permissions,
}

impl HostState {
    pub fn new(extension_id: String, permissions: greentic_ext_contract::describe::Permissions) -> Self {
        Self { extension_id, permissions }
    }
}

// Implement the host traits generated by bindgen:
// logging::Host, i18n::Host, secrets::Host, broker::Host, http::Host

impl extension_host::logging::Host for HostState {
    fn log(&mut self, level: extension_host::logging::Level, target: String, message: String) {
        use extension_host::logging::Level as L;
        let ext = &self.extension_id;
        match level {
            L::Trace => tracing::trace!(%ext, %target, "{message}"),
            L::Debug => tracing::debug!(%ext, %target, "{message}"),
            L::Info => tracing::info!(%ext, %target, "{message}"),
            L::Warn => tracing::warn!(%ext, %target, "{message}"),
            L::Error => tracing::error!(%ext, %target, "{message}"),
        }
    }

    fn log_kv(&mut self, level: extension_host::logging::Level, target: String,
              message: String, fields: Vec<(String, String)>) {
        // Simplified: format fields into the message.
        let pairs: Vec<String> = fields.iter().map(|(k, v)| format!("{k}={v}")).collect();
        let msg = if pairs.is_empty() { message } else { format!("{message} {{{}}}", pairs.join(", ")) };
        self.log(level, target, msg);
    }
}

impl extension_host::i18n::Host for HostState {
    fn t(&mut self, key: String) -> String {
        // Stub: return the key itself. Real i18n wired later.
        key
    }
    fn tf(&mut self, key: String, _args: Vec<(String, String)>) -> String {
        key
    }
}

impl extension_host::secrets::Host for HostState {
    fn get(&mut self, uri: String) -> Result<String, String> {
        // Check uri against allowlist
        if !self.permissions.secrets.iter().any(|allowed| uri.starts_with(allowed)) {
            return Err(format!("permission denied for secret: {uri}"));
        }
        // Stub: no secrets backend wired yet
        Err("no secrets backend configured".into())
    }
}

impl extension_host::broker::Host for HostState {
    fn call_extension(&mut self, kind: String, target_id: String,
                       function: String, _args_json: String) -> Result<String, String> {
        // Check caller has permission to call this kind
        if !self.permissions.call_extension_kinds.iter().any(|k| k == &kind) {
            return Err(format!("{} may not call {kind} extensions", self.extension_id));
        }
        // Stub: cross-ext dispatch not wired in 4B.0
        Err(format!("broker call to {target_id}.{function} not implemented in 4B.0"))
    }
}

impl extension_host::http::Host for HostState {
    fn fetch(&mut self, req: extension_host::http::Request)
             -> Result<extension_host::http::Response, String> {
        // Check url allowlist
        let allowed = self.permissions.network.iter().any(|pattern| {
            // Simple glob: pattern ends with /* and req.url starts with pattern minus /*
            let base = pattern.trim_end_matches("/*");
            req.url.starts_with(base)
        });
        if !allowed {
            return Err(format!("network permission denied for url: {}", req.url));
        }
        // Stub: http not wired in 4B.0
        Err("http fetch not implemented in 4B.0".into())
    }
}
```

## Task 3 — Instance builder + dispatch API

**Files:**
- Modify: `crates/greentic-ext-runtime/src/loaded.rs` + `src/runtime.rs`

- [ ] **Step 1: Add `LoadedExtension::build_instance` method**

Extend `LoadedExtension`:

```rust
use wasmtime::component::{Linker, Instance};
use wasmtime::Store;

impl LoadedExtension {
    pub fn build_store_and_instance(
        &self,
        engine: &wasmtime::Engine,
    ) -> anyhow::Result<(Store<HostState>, Instance)> {
        let mut linker: Linker<HostState> = Linker::new(engine);

        crate::host_bindings::greentic::extension_host::logging::add_to_linker(
            &mut linker, |s| s,
        )?;
        crate::host_bindings::greentic::extension_host::i18n::add_to_linker(
            &mut linker, |s| s,
        )?;
        crate::host_bindings::greentic::extension_host::secrets::add_to_linker(
            &mut linker, |s| s,
        )?;
        crate::host_bindings::greentic::extension_host::broker::add_to_linker(
            &mut linker, |s| s,
        )?;
        crate::host_bindings::greentic::extension_host::http::add_to_linker(
            &mut linker, |s| s,
        )?;

        let state = HostState::new(
            self.id.as_str().to_string(),
            self.describe.runtime.permissions.clone(),
        );
        let mut store = Store::new(engine, state);
        let instance = linker.instantiate(&mut store, &self.component)?;
        Ok((store, instance))
    }
}
```

- [ ] **Step 2: Add `ExtensionRuntime::invoke_tool` and sibling dispatch methods**

Add to `impl ExtensionRuntime`:

```rust
pub fn invoke_tool(
    &self,
    ext_id: &str,
    tool_name: &str,
    args_json: &str,
) -> Result<String, RuntimeError> {
    let loaded = self
        .loaded
        .load()
        .get(&crate::loaded::ExtensionId(ext_id.to_string()))
        .cloned()
        .ok_or_else(|| RuntimeError::NotFound(ext_id.to_string()))?;

    let (mut store, instance) = loaded
        .build_store_and_instance(&self.engine)
        .map_err(RuntimeError::Wasmtime)?;

    let tools_iface = instance
        .exports(&mut store)
        .instance("greentic:extension-design/tools@0.1.0")
        .ok_or_else(|| {
            RuntimeError::Wasmtime(anyhow::anyhow!("tools interface not exported"))
        })?;
    let invoke_tool = tools_iface
        .typed_func::<(String, String), (Result<String, ...>,)>("invoke-tool")
        .map_err(RuntimeError::Wasmtime)?;

    // ...
}
```

NOTE: the exact wasmtime Component type-func signature for `invoke-tool` returning `result<string, extension-error>` is tricky — `extension-error` is a variant. Easiest path: use the generated bindings (bindgen produces typed wrapper `ToolsPre::new(&instance, &mut store)?` with typed method).

Given the complexity, **simplified approach**: use `bindgen!` output to get typed host-side call methods. That replaces the manual `typed_func` dance.

Adjust host_bindings.rs to `{ async: false, with: { /* ... */ }, ... }` and use the generated `DesignExtension::instantiate(...)` pattern.

If the generated API turns out to be awkward, fall back to untyped dispatch via `instance.get_func()` + manual val conversion. Note any deviation as BLOCKED or DONE_WITH_CONCERNS.

```rust
use std::path::Path;

use greentic_ext_runtime::{DiscoveryPaths, ExtensionRuntime, RuntimeConfig};

#[tokio::test]
async fn invoke_validate_card_on_ac_extension() {
    // Assumes AC ext has been built and .gtxpack exists
    let pack = Path::new(env!("CARGO_MANIFEST_DIR"))
        .parent().unwrap().parent().unwrap()
        .join("reference-extensions/adaptive-cards/greentic.adaptive-cards-1.6.0.gtxpack");

    if !pack.exists() {
        eprintln!("skipping: AC .gtxpack not built. run reference-extensions/adaptive-cards/build.sh first");
        return;
    }

    // Unpack into a temp home
    let tmp = tempfile::TempDir::new().unwrap();
    let ext_dir = tmp.path().join("extensions/design/greentic.adaptive-cards-1.6.0");
    greentic_ext_testing::unpack_to_dir(&pack, &ext_dir).unwrap();

    let config = RuntimeConfig::from_paths(DiscoveryPaths::new(tmp.path().to_path_buf()));
    let mut rt = ExtensionRuntime::new(config).unwrap();
    rt.register_loaded_from_dir(&ext_dir).unwrap();

    // Valid card
    let result = rt.invoke_tool(
        "greentic.adaptive-cards",
        "validate_card",
        r#"{"card":{"type":"AdaptiveCard","version":"1.6"}}"#,
    ).expect("valid card should succeed");

    let parsed: serde_json::Value = serde_json::from_str(&result).unwrap();
    assert_eq!(parsed["valid"], serde_json::Value::Bool(true));
}
```

## Risks

This phase touches wasmtime Component Model APIs directly. Known risks:
- `bindgen!` macro may need specific WIT world layout that `runtime-side.wit` doesn't fulfill cleanly
- Variant type (`extension-error`) host-side representation may be verbose
- Instance-per-call allocation may be slow; can optimize with instance pool later

If **blocked** by wasmtime API quirks, fall back to stub-path: `invoke_tool` returns `Err(RuntimeError::Wasmtime(anyhow!("dispatch layer: stub in 4B.0; full wiring in 4B.0.1")))`. Designer refactor can still proceed against the method signature; end-to-end AC call test gets marked `#[ignore]`.
